## 项目主要需求

### 项目概述

【全栈Web应用】多仓库库存管理系统。面向中小型仓储企业，支持多仓库/多库位的精细化库存管理，覆盖入库、出库、盘点、调拨全流程。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿全栈）

1. **仓库（Warehouse）** — 字段：仓库名称、仓库编码、地址、面积（平方米）、管理员ID、状态（Active/Inactive/Maintenance）、联系电话
   - 关联：1个仓库 → 多个库位（Shelf：货架编号、层数、列数、容量）
   - 业务动作：创建仓库、维护库位、启用/停用
   - **贯穿全栈**

2. **商品（Product）** — 字段：商品名称、SKU、分类ID、规格型号、单位、重量、体积、条形码、最低库存、最高库存、保质期天数（可选）、商品图片URL
   - 关联：1个商品 → 多条库存记录（按仓库+库位）
   - 业务动作：创建商品、更新信息、设置库存阈值
   - **贯穿全栈**

3. **库存记录（StockRecord）** — 字段：商品ID、仓库ID、库位ID、批次号、数量、入库日期、过期日期（可选）、最近操作类型（入库/出库/盘点/调拨）、最近操作时间
   - 业务动作：入库增加、出库扣减、盘点校准、调拨转移
   - **贯穿全栈**

4. **出入库单（StockOrder）** — 字段：单号（自动生成）、类型（Inbound/Outbound/Transfer/InventoryCheck）、来源仓库ID（调拨时）、目标仓库ID、状态（Draft/Submitted/Processing/Completed/Cancelled）、创建人ID、审批人ID、备注、明细列表
   - 关联：1个单据 → 多个明细项（StockOrderItem：商品ID、库位ID、数量、实际数量）
   - **贯穿全栈**

5. **商品分类（Category）** — 字段：分类名称、父分类ID（支持多级）、排序、图标
   - **贯穿全栈**

### 核心页面（≥ 4 个）

1. **库存总览仪表盘** `/dashboard` — 消费：Warehouse + StockRecord + StockOrder
   - 各仓库库存金额柱状图、库存预警商品TOP10、今日出入库流水数、库存周转率
   - 使用 `<StatusBadge>`, `<StockAlert>` 共享组件

2. **仓库管理** `/warehouses` — 消费：Warehouse + StockRecord
   - 仓库列表（卡片式展示面积和库存量），仓库详情页含库位地图（网格布局展示每个库位的占用率）
   - 使用 `<StatusBadge>`, `<OccupancyBar>`

3. **商品管理** `/products` — 消费：Product + Category + StockRecord
   - 商品列表（树形分类 + 搜索 + 筛选），商品详情（各仓库库存分布表）
   - 使用 `<StatusBadge>`, `<StockAlert>`

4. **出入库管理** `/orders` — 消费：StockOrder + StockOrderItem + Product
   - 单据列表（按类型Tab切换），创建出/入库单（动态添加明细行），单据详情（含审批流程）
   - 使用 `<StatusBadge>`, `<EmptyState>`

5. **盘点管理** `/inventory-check` — 消费：StockRecord + Warehouse
   - 选择仓库和库位范围，逐商品录入实盘数量，自动计算差异，生成盘点报告

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / WarehouseManager / Operator / Viewer
   - 触达文件层：DB role表 + RBAC中间件 + 路由守卫 + 按钮显隐（Viewer只读）+ 类型定义

2. **操作日志**
   - 记录所有出入库、盘点、调拨操作的完整流水
   - 触达文件层：DB operation_log表 + 日志中间件 + service埋点 + 前端日志页面 + 类型定义

3. **全局异常处理**
   - 触达文件层：后端异常过滤器 + controller业务异常 + 前端ErrorBoundary + API拦截器 + 错误码枚举

### 共享枚举/常量（≥ 2 组）

1. **单据类型 OrderType**：Inbound / Outbound / Transfer / InventoryCheck
   - 后端：`backend/src/types/enums.ts`；前端：`frontend/src/types/enums.ts`
   - README 列出所有出现位置

2. **单据状态 OrderStatus**：Draft / Submitted / Processing / Completed / Cancelled

3. **仓库状态 WarehouseStatus**：Active / Inactive / Maintenance

4. **库存操作类型 OperationType**：Inbound / Outbound / Adjust / Transfer

### 共享前端组件清单（≥ 3 个）

1. **`<StatusBadge>`** — 全部页面
2. **`<StockAlert>`** — 仪表盘、商品管理（库存低于阈值时红色高亮）
3. **`<OccupancyBar>`** — 仓库管理（库位占用率进度条）
4. **`<EmptyState>`** — 所有列表空状态

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useStockQuery`** — 封装库存查询（按仓库/商品/批次组合筛选）
2. **`usePagination`** — 通用分页
3. **`formatStockNumber`** — 库存数量格式化（含单位）

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色校验
3. **`auditLogMiddleware`** — 操作日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`warehouse-stock`
- **前端端口**：18705 / 80
- **后端端口**：19205 / 3000
- **前端**：Vue 3 + TypeScript + Vite
- **UI 库**：Element Plus
- **图表**：ECharts
- **后端**：Express + TypeScript
- **数据库**：MySQL 8.0
- **ORM**：Prisma
- **认证**：JWT
- **状态管理**：Pinia
- **构建工具**：Vite

### 部署要求

Docker Compose 一键部署。

docker-compose.yml 要求：
- 顶层 `name: warehouse-stock`
- 不写 `version:` 字段
- 容器名带前缀
- MySQL 命名卷 + healthcheck
- 后端 depends_on service_healthy
- 前端 `${FRONTEND_PORT:-80}:80`，后端 `${BACKEND_PORT:-3000}:3000`

### Git 要求

`git init` + `.gitignore`。
